Extras

Extras provide enhancements that might not be necessary for every project. Therefore, they are not included as part of the default Tyfy stylesheet. If useful for your project, they can be @included in your SCSS file after the other Tyfy modules. The available extras are:

  • Icons
  • Drupal Styles
  • Responsive Text

Icons

@include "[path to tyfy]/source/scss/extras/icons"

Because Tyfy only needs a handful of icons, I've included a dozen or so common icons as a custom web font rather than recommend an existing font library.

Node.js and the webfont package are used to convert a set of .svg icons into an icon webfont called tyfyicons. The Sketch file used to create the .svgs is included in the git repository for Tyfy.

The tyfyicons font is not necessarily required. If you want to use a different icon font, you will need to customize the values for the icon character code variables in the required/settings module to use character values for the selected font. Tyfyicons are used in styles for lists, blockquotes, collapsables, and tabs. If you are not planning to use those modules, the icon font module can be omitted and the character code variables ignored.

Using Icons in SCSS

Because of an issue with SASS, the icon font character codes are stored in the required/settings without their preceding slash:

$tyfyicons-angle-up:"ea05";

To properly apply the character codes in the CSS content property, Tyfy has a custom function named charcode that accepts a character code variable and returns the correct character code string.

Drupal Styles

@include "[path to tyfy]/source/scss/extras/drupal"

Much of the front-end development work that I perform is for Drupal. Therefore, it is useful to me to have a set of Tyfy-based styles to use with Drupal. This module provides the following Drupal style overrides:

  • styles for status message
  • styles for primary tabs
  • styles to identify unpublished node pages
  • styles for invisible and hidden elements
  • removal of Drupal's custom menu list item bullets
  • removal of default block margins
  • proper styling for the admin module menu when using Tyfy

Variables

$status-default-color
Color for status messages; !default value is #33aa33
$status-warning-color
Color for warning messages; !default value is #aa8833
$status-error-color
Color for error messages; !default value is #aa3333

Responsive Text

@include "[path to tyfy]/source/scss/extras/responsive_text"

Some responsive website designs will vary the text sizes depending on the screen width. On mobile, it may be desirable to fit more text into the limited horizontal space; on large desktop screens, larger text may be easier to read and more comfortably fill the screen. This module adjusts text sizes according to viewing context. The elements for which text is adjusted are:

  • <body> and all child elements for which a font-size override has not been defined
  • <h1> to <h6>, including collapsable headings
  • <pre>, <code>, <kdb>, and <samp>; these elements have a font size override in the normalize module and are not affected by the font-size changes to <body>

Variables

Several new variables are defined in this module. There are no variables for the responsive heading sizes. Those sizes are based on the default modular scale with scale step decreases for small screen headings and scale step increases for larger screen headings.

$tyfy-break-font-small
Breakpoint below which text sizes are reduced; !default value is 40em
$tyfy-break-font-large
Breakpoint above which text sizes are increased; !default value is 80em
$tyfy-font-size-base-small
The base font size for small screens; !default value is $tyfy-font-size-base * 0.9
$tyfy-font-size-base-large
The base font size for large screens; !default value is $tyfy-font-size-base / 0.9

Example

These documentation pages utilize the responsive text module. You can adjust the size of your browser window to see the changes to the font sizes.
