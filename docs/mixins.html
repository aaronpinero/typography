Tyfy includes several SASS mixins that facilitate the use of the framework and provide commonly used styles and effects.

@mixin button

Display a link or other tag as a button. The text color of the button is the default background color. To apply the button class to a link, use the [href] attribute selector. See the links and utility modules documetation for more information.

Arguments

$color_default
Default button color; must be rgb, named color, or hexidecimal
Default value is $tyfy-color-link-default
$color_active
Hovered and active button color; must be rgb, named color, or hexidecimal
Default value is $tyfy-color-link-active
$border_radius
Border radius of the button; all unit values allowed
Default value is tygrid(0.25)

Examples

tag.button {
  @include button;
}
a.button[href] {
  @include button(red,blue,0px);
}

@mixin clearfix

Clear floats within an element. This mixin does not accept any arguments. Code comes from HTML5 Boilerplate.

Example

tag.class {
  @include clearfix;
}

@mixin fade-in

Apply a non-looping fade in animation to an element with the given animation duration.

Arguments

$duration
Duration in seconds or milliseconds
Default value is 1000ms

Example

tag.class {
  @include fade-in(2s);
}

@mixin mod-colors

Change the framework’s colors within a specific context. There are no default values for this mixin; all color values must be specified. This mixin is used in the reverse colors module.

Arguments

All values are colors; must be rgb, named color, or hexidecimal

$bg
Background color for this context
$body
Text color for this context
$head
Text color of any headings in this context
$light
Text color for the light class in this context
$quote
Text color for the pullquote class in this context
$default
Link and button color in this context
$active
Active link and button color in this context
$border
Border color in this context
$shade
Background shade color in this context

Example

From the reverse colors module.

// reverse colors
$tyfy-color-background-reverse:     #333 !default;
$tyfy-color-text-body-reverse:      #eee !default;
$tyfy-color-text-head-reverse:      #eee !default;
$tyfy-color-text-light-reverse:     mix($tyfy-color-text-body-reverse,$tyfy-color-background-reverse,45%) !default;
$tyfy-color-text-pullquote-reverse: #cccc33 !default;
$tyfy-color-link-default-reverse:   #91bbe4 !default;
$tyfy-color-link-active-reverse:    #dc72a7 !default;
$tyfy-color-border-reverse:         mix($tyfy-color-text-body-reverse,$tyfy-color-background-reverse,10%) !default;
$tyfy-color-shade-reverse:          mix($tyfy-color-text-body-reverse,$tyfy-color-background-reverse,5%) !default;

// class
.reverse {
  @include mod-colors(
    $tyfy-color-background-reverse,
    $tyfy-color-text-body-reverse,
    $tyfy-color-text-head-reverse,
    $tyfy-color-text-light-reverse,
    $tyfy-color-text-pullquote-reverse,
    $tyfy-color-link-default-reverse,
    $tyfy-color-link-active-reverse,
    $tyfy-color-border-reverse,
    $tyfy-color-shade-reverse
  );
}
@mixin set-sibling-margins
Arguments
$context
Allowed values are all, headings, or lists
Default value is all
$measure
Allowed values are rem-unit lengths
Default value is $tyfy-spacing-standard
Purpose
Set the size of adjacent sibling margins in a given context. The all context sets the value for all tags; the headings context sets the value only for headings; the lists context sets the value only inside of ordered, unordered, and definition lists. For more information, see the Margins documentation.
Example
tag.class {
  @include set-sibling-margins(all,tygrid(1));
}
@mixin sibling-margins
Arguments
$state
Allowed values are on or off
Default value is on
Purpose
Turn adjacent sibling margins on or off within a specific context. If the state is on, the margins are set to the Tyfy default values. To override with specific values, use the set-sibling-margins mixin.
Example
tag.class {
  @include sibling-margins(off);
}
@mixin visuallyhidden
Arguments
$state
Allowed values are true or false
Default value is true
Purpose
Visually hide an element, but allow screen readers and search engines to read the element’s content. The default true argument hides the element. The false argument is used to override the default styles. Code is adapted from HTML5 Boilerplate.
Example
tag.hide-me {
  @include visuallyhidden;
  &.show-me {
    @include visuallyhidden(false);
  }
}
