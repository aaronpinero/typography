Link Styles

Links must have a style that is different from body text, but anchors should not. Because anchors and links share the <a> tag, it is necessary to progressively define styles for this tag. Here, the default appearance of the <a> tag is the same as for body text. Using an attribute selector, those elements that have an href attribute have link styles applied.

a {
  [styles]
  &[href] {
    [link styles]
    &:hover,
    &:active {
      [hovered and active link styles]
    }
  }
}

For links, I have used border-bottom to apply an underline instead of text-decoration. This way, the underline can be styled and transitions can be employed. The framework includes SASS variables to globally define transition speeds.

Links in headings have a slightly different appearance, as I prefer links in this context to have a default appearance that is not too far removed from the heading style.

There are contexts in which I prefer links to not have any underline. When creating tabs, menus, or other navigation, I often must override the default style to remove the underline. The default link styles do not have an underline. The links-underlined class can be applied where links should be presented with an underline.

Links can be presented as a button using the button and button-outline classes. These classes are dependent on mixins of the same name. Using mixins allows the same button styling to be applied to other elements.
