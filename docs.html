About Tyfy

Tyfy is a framework for applying elements of typographic style to front-end development. I originally named this project “typography” for my effort to encode typographic rules and patterns. The name later became a short form of the made-up verb “typogrify” — the action of applying typographic style. So “Tyfy” rhymes with “WiFi” or “Sci-Fi”.

For more information about how Tyfy came to be, you can read the Medium article.

Rationale

There are three reasons why Tyfy was created:

  • To collect all of my best design and pattern work into one system.
  • To create a system that I can employ in future projects.
  • To learn something along the way.

A framework is an expression of the values, preferences, priorities, and experience of the person or persons creating it. While I embrace some existing frameworks (Bootstrap 4, PureCSS), I wanted to create something that reflects my values and the work that I do.

Build Tools

This project uses node.js for assembling the icon webfont, compiling the SCSS files into CSS, and for assembling the documentation pages. I use npm scripts to automate these tasks.

> npm run build-css
> npm run watch-css
Creates compressed CSS. The watch script uses nodemon to check for changes to the SCSS files.
> npm run build-css-dev
> npm run watch-css-dev
Creates readable CSS with source file comments. The watch script uses nodemon to check for changes to the SCSS files.
> npm run build-icons
Creates the webfont and SCSS files for the icons.
> node docs.js
Compiles documentation files using templatesjs to assemble the HTML pages and their includes.
