Get Started

Tyfy is a framework for applying elements of typographic style to your front-end development. It provides a baseline grid, modular scale, sensible margins, and styles for common HTML elements and content patterns.

To get started you can either use the compiled CSS or begin with the SCSS files and customize the framework by overriding the default variable values. The latter approach is recommended as it allows you to take full advantage of the mixins and functions for the baseline grid and modular scale.

Start with SCSS (recommended)

Start with CSS

The compiled CSS includes all of the required, essential, and optional Tyfy modules. It also adds the icons module from the extras collection. Use this approach if you want to get started quickly and don't need any customization.

  1. Clone the github project or install using npm install tyfy
  2. Within the tyfy project folder, the dist directory contains the framework's compiled stylesheet, javascripts, and icon font. Move the dist directory into your project folder.

  3. Add the Tyfy stylesheet to the your HTML <head>.

    <head>
  // head tags
  <link rel="stylesheet" href="dist/style/tyfy.css">
</head>

  4. Add jQuery (1.12.4 or above) and the Tyfy JavaScript files to the end of your HTML <body>.

    <body>
  // content of page
  <script src="http://code.jquery.com/jquery-1.12.4.min.js"></script>
  <script src="dist/script/collapsables.js"></script>
  <script src="dist/script/tables.js"></script>
  <script src="dist/script/tabs.js"></script>
</body>
  5. Now you’re ready to go. Check out the modules documentation to learn more about Tyfy styles and CSS classes.

Modular Scale

Tim Brown, Head of Typography at Adobe Typekit and co-developer of modularscale.com writes:

“By basing the dimensions of our compositions on values from modular scales, we can achieve a visual harmony not found in layouts that use arbitrary, conventional, or easily divisible numbers.”

More simply, Robert Bringhurst writes “don’t compose without a scale”. Is he talking about music or typgraphic design?

Tyfy follows Bringhurst’s advice and uses a modular scale for font size values.

Selecting a Modular Scale

The modular scale for your project depends on the value of two Tyfy variables: a base and a scalar (or ratio). The default value of these variables is defined in the core module.

$tyfy-font-size-base: 1rem !default;
$tyfy-font-size-scalar: 1.2 !default;

The base of the scale is $tyfy-font-size-base (this is also the default font size for the <body> in Tyfy). Every step in the modular scale of font size values is calculated by multiplying the base value by ($tyfy-font-size-scalar)n, where n is the number of steps above or below the base value. The ratio between any two adjacent steps in the scale is the value of the scalar.

Using a modular scale for font sizes not only provides the visual harmony that Tim Brown described above, but also consistent and effective visual contrast between elements with different text sizes. The modular scale also takes the mystery out of how to select font sizes for elements like headings, subheadings, and pull quotes.

The tyscale Function

Tyfy provides a modular scale function that makes it easy to assign font size values based the modular scale. No math required! The function is named tyscale and is defined in the functions module.

@function tyscale($exponent) {
  $i:0;
  $scale:$tyfy-font-size-base;
  @if $exponent < 0 { // negative values step down
    @while $i > $exponent {
      $scale:$scale/$tyfy-font-size-scalar;
      $i:$i - 1;
    }
  }
  @else if $exponent > 0 { // positive values step up
    @while $i < $exponent {
      $scale:$scale*$tyfy-font-size-scalar;
      $i:$i + 1;
    }
  }
  @return $scale;
}

The tyscale function accepts an integer argument and returns a font size value. The integer argument specifies the number of steps up or down the modular scale from the base font size.

This function is used in Tyfy to set all font size values. For example:

small {
  font-size: tyscale(-1) !default; // 0.833rem; one scale step down from the base size
}
h6 {
  font-size: tyscale(0) !default;  // 1rem; using 0 as the exponent is the same as setting the value to the base size
}
h5 {
  font-size: tyscale(1) !default;  // 1.2rem; one scale step up from the base size
}
h4 {
  font-size: tyscale(2) !default;  // 1.44rem; two scale steps up from the base size
}
[...]

Since all font sizes in Tyfy are based on the modular scale, you can quickly change every element’s font size by modifying either the base size or the scalar value. In this way you can fine-tune the typographic design of your project.

Paragraphs

Paragraphs inherit font size and line height from the <body>. Tyfy provides CSS classes for common style variations.

Classes

<p class="lede">
Display a paragraph as a lede. Font size of a lede is equal to $tyfy-font-size-lede.
<p class="justified">
Display a paragraph with justified text.
<tag class="paragraphs-justified">
Display all child paragraphs with justified text.
<tag class="cols-2">
Display content in two columns. The style is responsive and degrades to a single column when the screen width is less than the value of $tyfy-break-cols-2.

Variables

$tyfy-break-cols-2: 50em !default;
The breakpoint above which text inside an element with the cols-2 class is displayed in two columns.
$tyfy-font-size-lede: tyscale(2) !default;
The font size applied to a paragraph with the lede class.

Headings

The font sizes for heading levels 1 through 6 are based on the modular scale. Heading font sizes are also available as SASS variables and CSS classes.

To emphasize the meaning of a heading, it should be positioned closer to the text that follows than the text preceding. Adjacent sibling selectors are used to define the margins for headings in context of other elements. If the element preceding a heading is not also a heading, the top margin for the heading will be larger than the standard margin. In CSS, this is accomplished as follows:

* + * {
  margin-top: $tyfy-spacing-standard;
}
:not(h1) + h2 {
  margin-top: $tyfy-spacing-large;
}

Classes

<tag class="font-size-h[x]">
Display an element using the font size of heading level x.

Variables

$tyfy-font-size-h6: tyscale(0) !default; $tyfy-font-size-h5: tyscale(1) !default; $tyfy-font-size-h4: tyscale(2) !default; $tyfy-font-size-h3: tyscale(3) !default; $tyfy-font-size-h2: tyscale(4) !default; $tyfy-font-size-h1: tyscale(6) !default;
The font size for heading levels 1 to 6.
$tyfy-spacing-large: tygrid(3) !default
The top margin of headings when the preceding element is not another heading.
$tyfy-font-heading: $tyfy-serif !default
The font family for headings.
$tyfy-color-text-head: mix($tyfy-color-foreground,$tyfy-color-background,95%) !default;
The text color for headings.

About Tyfy

Tyfy is a framework for applying elements of typographic style to your front-end development.

I first named this project “typography” for my effort to encode typographic rules and patterns. The name later changed to a short form of the portmanteau “typogrify”. So “Tyfy” rhymes with “WiFi” or “Sci-Fi”.

For more information about how Tyfy came to be, you can read the Medium article.

Rationale

There are three reasons why Tyfy was created:

  • To collect all of my best design and pattern work into one system.
  • To create tolls that enhance my front-end development workflow.
  • To learn something along the way.

A framework is an expression of the values, preferences, priorities, and experience of the person or persons creating it. While I embrace some existing frameworks (Bootstrap 4, PureCSS), I wanted to create something that reflects my values and the work that I do.

Thanks

This project would not have happened without the benefit of work done by others. Their contributions to front-end development are part of my experience, and so my work reflects theirs. Tyfy has something old, something new, something borrowed, and something blue from the following sources:

  • HTML5 Boilerplate
  • Archetype
  • Bootstrap
  • PureCSS
  • Simpleicons
  • Better Web Type

Build Tools

This project uses node.js for assembling the icon webfont, compiling the SCSS files into CSS, and for assembling the documentation pages. These tasks are automated with npm scripts.

> npm run build-css
> npm run watch-css
Uses node-sass to create compressed CSS files. Autoprefixing is accomplished with autoprefixer and postcss-cli. The watch script uses nodemon to check for changes to the SCSS files.
> npm run build-css-dev
> npm run watch-css-dev
Uses node-sass to create readable CSS with source file comments. Files are linted prior to compilation using stylelint. Autoprefixing is accomplished with autoprefixer and postcss-cli. The watch script uses nodemon to check for changes to the SCSS files.
> npm run build-icons
Uses webfont the webfont and SCSS files for the icons.
> node docs.js
Compiles documentation files using templatesjs to assemble the HTML pages and their includes.
